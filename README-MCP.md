# BaseRadar MCP Server Setup Guide

> **Target Audience**: Users with any programming background  
> **Compatible Clients**: Any MCP-compatible client (Claude Desktop, Cline, Continue, Cherry Studio, etc.)

---

## üì• Step 1: Get Project Code

Why do you need to get the project code?

The AI analysis functionality needs to read news data from the project to work. Whether you use GitHub Actions or Docker deployment, the news data generated by the crawler is saved in the project's `output` directory. Therefore, before configuring the MCP server, you need to get the complete project code (including data files).

Based on your technical level, you can choose any of the following methods:

### Method 1: Git Clone (Recommended for Technical Users)

If you're familiar with Git, you can use the following command to clone the project:

```bash
git clone https://github.com/your-username/your-project-name.git
cd your-project-name
```

**Advantages**:
- Can update latest data to local with one command (`git pull`)

### Method 2: Direct Download ZIP (Recommended for Beginners)

1. **Visit GitHub Project Page**
   - Project link: `https://github.com/your-username/your-project-name`

2. **Download ZIP**
   - Click the green "Code" button
   - Select "Download ZIP"
   - Or directly visit: `https://github.com/your-username/your-project-name/archive/refs/heads/master.zip`

**Notes**:
- Steps are slightly more cumbersome, subsequent data updates require repeating the above steps, then overwriting local data (output directory)

## üîß Step 3: Configure MCP Client

The MCP server supports two transport modes:

### STDIO Mode (Recommended for Local Use)

This mode uses standard input/output for communication, suitable for local MCP clients.

**Configuration Steps**:

1. **Open your MCP client settings**
   - Claude Desktop: Settings ‚Üí Developer ‚Üí Edit Config
   - Cline: Settings ‚Üí MCP Servers
   - Continue: Settings ‚Üí MCP
   - Cherry Studio: Settings ‚Üí MCP ‚Üí Add Server

2. **Add MCP Server**
   - **Name**: BaseRadar (or any name you prefer)
   - **Type**: STDIO
   - **Command**: [Full path to UV executable]
   - **Arguments**: `--directory [project path] run python mcp_server/server.py`

   Example configuration:
   ```json
   {
     "mcpServers": {
       "BaseRadar": {
         "command": "C:\\Users\\YourName\\.cargo\\bin\\uv.exe",
         "args": [
           "--directory",
           "D:\\2025\\Blockchain\\basechain\\BaseRadar",
           "run",
           "python",
           "mcp_server/server.py"
         ]
       }
     }
   }
   ```

3. **Save and Enable**
   - Click **Save**
   - Ensure the MCP server switch in the list is **ON** ‚úÖ

### HTTP Mode (Recommended for Remote Access)

This mode uses HTTP for communication, suitable for remote access or multiple client sharing.

**Start HTTP Server**:

#### Windows
Double-click `start-http.bat`

#### Mac/Linux
```bash
./start-http.sh
```

**Configure Client**:

1. **Open your MCP client settings**

2. **Add MCP Server**
   - **Name**: BaseRadar
   - **Type**: HTTP or streamableHttp
   - **URL**: `http://localhost:3333/mcp`

   Example configuration:
   ```json
   {
     "mcpServers": {
       "BaseRadar": {
         "url": "http://localhost:3333/mcp",
         "transport": "http"
       }
     }
   }
   ```

## üéØ Advanced Configuration

### Custom Project Root

If your project is in a different location, you can specify it:

```bash
uv run python mcp_server/server.py --project-root /path/to/your/project
```

### Custom HTTP Port

If port 3333 is already in use:

```bash
uv run python mcp_server/server.py --transport http --port 8080
```

Then update your client configuration to use `http://localhost:8080/mcp`

## üìö Available Tools

The MCP server provides 13 tools:

### Basic Data Query (P0 Core)
1. `get_latest_news` - Get latest news
2. `get_news_by_date` - Query news by date (supports natural language)
3. `get_trending_topics` - Get trending topics

### Intelligent Search Tools
4. `search_news` - Unified news search (keyword/fuzzy/entity)
5. `search_related_news_history` - Historical related news search

### Advanced Data Analysis
6. `analyze_topic_trend` - Unified topic trend analysis (heat/lifecycle/viral/predict)
7. `analyze_data_insights` - Unified data insights analysis (platform compare/activity/keyword co-occurrence)
8. `analyze_sentiment` - Sentiment analysis
9. `find_similar_news` - Similar news search
10. `generate_summary_report` - Daily/weekly summary generation

### Configuration and System Management
11. `get_current_config` - Get current system configuration
12. `get_system_status` - Get system running status
13. `trigger_crawl` - Manually trigger crawl task

## üîç Troubleshooting

### Server Not Starting

- Check if Python and UV are installed correctly
- Verify the project path is correct
- Check if port 3333 is available (for HTTP mode)

### Tools Not Available

- Ensure the MCP server is enabled in your client
- Check if the server process is running
- Verify configuration file format is correct

### No Data Returned

- Check if `output` directory contains data files
- Verify date range is correct
- Ensure platforms are configured in `config/config.yaml`
